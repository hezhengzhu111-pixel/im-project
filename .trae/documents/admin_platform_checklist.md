# 后台管理平台及日志管理模块上线检查清单 (Checklist)

## 1. 基础设施检查
- [ ] 确认 Elasticsearch 节点运行正常，健康状态为 green。
- [ ] 确认 Kafka Broker 及 Zookeeper 运行正常，Topic `im-service-logs` 已创建且分区配置正确。
- [ ] 确认 Filebeat 在各微服务节点均已部署并能正常采集 `.log` 文件。
- [ ] 确认 Nacos 配置中心已添加 `log-service` 和 `admin-service` 所需的配置。

## 2. 核心服务部署
- [ ] 确认 `admin-service` 已成功启动，接口可正常响应 (包括网关路由配置完成)。
- [ ] 确认 `log-service` 已成功启动，Kafka 消费者正常订阅消息，SSE 接口 `/api/logs/stream` 可连接。

## 3. 功能验证
- [ ] **全链路 TraceId**：在前端发起任意业务请求，验证网关至各微服务的请求头中是否均携带 `X-Trace-Id`。
- [ ] **多线程日志**：验证在 `@Async` 或虚拟线程中输出的日志是否依然带有原始 `TraceId`。
- [ ] **日志脱敏**：故意打印包含手机号或身份证号的日志，验证输出到文件及推送到前端的日志是否已替换为 `****`。
- [ ] **实时推送 (SSE)**：打开前端日志监控页面，触发业务操作，验证 1 秒内是否在页面上可见新产生的日志。
- [ ] **ES 聚合查询**：使用特定的 `TraceId` 在前端面板或 Swagger 发起查询，验证是否能一次性返回完整链路日志。
- [ ] **降级策略验证**：关闭 Elasticsearch 容器，发起日志查询，验证系统是否能自动退化为读取本地日志文件并返回结果。

## 4. 性能与安全验证
- [ ] 运行 JMeter 压测脚本 `admin_platform_jmeter.jmx`，确保日志写入吞吐 ≥ 5000 条/秒。
- [ ] 确认 `traceId` 字段在 ES 中被正确映射为 `keyword` 类型，验证 P99 查询延迟是否 ≤ 200 ms。
- [ ] 确认前端访问 `admin-service` 接口时，鉴权拦截器能够正常拦截未授权请求。

## 5. 交付物与文档
- [ ] Swagger / OpenAPI 3.0 接口文档可正常访问 (`/v3/api-docs`)。
- [ ] 单元测试覆盖率报告生成完成并达标 (≥ 80%)。
- [ ] 确认 `docker-compose.yml` 可以一键启动全部所需大数据与监控组件。
