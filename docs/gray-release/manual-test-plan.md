## 测试用例

### TC-01: 用户注册与登录

**优先级**: Critical
**前置条件**: 无

**测试步骤**:

1. 打开 Web 应用
2. 点击"注册"按钮
3. 输入用户名、密码
4. 提交注册表单
5. 验证注册成功
6. 使用刚注册的账号登录
7. 验证登录成功，进入主界面

**预期结果**:

- [ ] 注册成功，无错误提示
- [ ] 登录成功，进入聊天列表
- [ ] 用户头像/昵称显示正确

---

### TC-02: 个人信息编辑

**优先级**: High
**前置条件**: 已登录

**测试步骤**:

1. 进入"设置" -> "个人资料"
2. 修改昵称
3. 修改签名
4. 修改头像（上传图片）
5. 保存修改
6. 退出并重新登录
7. 验证修改已保存

**预期结果**:

- [ ] 所有字段修改成功
- [ ] 头像上传成功，显示正确
- [ ] 重新登录后修改保持

---

### TC-03: 好友添加与管理

**优先级**: Critical
**前置条件**: 两个已注册用户 A、B

**测试步骤**:

1. 用户 A 搜索用户 B
2. 用户 A 发送好友请求
3. 用户 B 查看好友请求列表
4. 用户 B 接受好友请求
5. 验证 A、B 好友列表互相对方
6. 用户 A 给用户 B 设置备注
7. 验证备注显示正确

**预期结果**:

- [ ] 搜索结果正确
- [ ] 好友请求发送成功
- [ ] 接受请求成功
- [ ] 好友列表正确显示
- [ ] 备注功能正常

---

### TC-04: 私聊消息（非 E2EE）

**优先级**: Critical
**前置条件**: 两个已登录且互为好友的用户 A、B

**测试步骤**:

1. 用户 A 打开与用户 B 的聊天窗口
2. 用户 A 发送文本消息
3. 验证用户 B 收到消息
4. 用户 B 回复消息
5. 验证用户 A 收到回复
6. 验证消息历史记录
7. 用户 A 撤回消息
8. 验证消息已撤回

**预期结果**:

- [ ] 消息发送成功
- [ ] 消息实时接收
- [ ] 历史记录正确
- [ ] 撤回功能正常

---

### TC-05: 私聊消息（E2EE）

**优先级**: Critical
**前置条件**: 两个已登录且互为好友的用户 A、B，已启用 E2EE

**测试步骤**:

1. 用户 A 打开与用户 B 的聊天窗口
2. 验证 E2EE 加密标识显示
3. 用户 A 发送加密文本消息
4. 验证用户 B 收到并解密消息
5. 用户 B 回复加密消息
6. 验证用户 A 收到并解密消息
7. 在数据库中验证消息已加密（非明文）

**预期结果**:

- [ ] E2EE 标识正确显示
- [ ] 消息加密传输
- [ ] 消息正确解密显示
- [ ] 数据库中无明文

---

### TC-06: 群聊创建与管理

**优先级**: Critical
**前置条件**: 至少三个用户 A、B、C

**测试步骤**:

1. 用户 A 创建群聊
2. 添加用户 B、C 到群聊
3. 验证群成员列表
4. 用户 A 修改群名称
5. 验证群名称更新
6. 用户 A 转让群主给用户 B
7. 验证群主已转让

**预期结果**:

- [ ] 群聊创建成功
- [ ] 成员添加成功
- [ ] 群信息修改成功
- [ ] 群主转让成功

---

### TC-07: 群聊消息

**优先级**: Critical
**前置条件**: 已创建的群聊，包含用户 A、B、C

**测试步骤**:

1. 用户 A 在群聊中发送消息
2. 验证用户 B、C 收到消息
3. 用户 B 回复消息
4. 验证用户 A、C 收到消息
5. 查看群聊历史记录
6. 验证消息已读状态

**预期结果**:

- [ ] 群消息发送成功
- [ ] 所有成员收到消息
- [ ] 历史记录正确
- [ ] 已读状态正确

---

### TC-08: 群聊 E2EE

**优先级**: Critical
**前置条件**: 已创建的群聊，已启用群 E2EE

**测试步骤**:

1. 验证群 E2EE 标识显示
2. 用户 A 发送加密群消息
3. 验证用户 B、C 收到并解密消息
4. 验证 Sender Key 分发正确
5. 新成员加入后验证可以解密新消息

**预期结果**:

- [ ] E2EE 标识正确显示
- [ ] 群消息加密传输
- [ ] 所有成员正确解密
- [ ] 新成员兼容性正常

---

### TC-09: 文件上传与下载

**优先级**: High
**前置条件**: 已登录用户

**测试步骤**:

1. 在聊天窗口点击"发送文件"
2. 选择图片文件上传
3. 验证上传成功，图片显示正确
4. 在聊天窗口点击"发送文件"
5. 选择文档文件上传
6. 验证上传成功
7. 下载已发送的文件
8. 验证文件完整性

**预期结果**:

- [ ] 图片上传成功，显示正确
- [ ] 文档上传成功
- [ ] 下载功能正常
- [ ] 文件内容完整

---

### TC-10: 朋友圈功能

**优先级**: High
**前置条件**: 已登录用户，有好友

**测试步骤**:

1. 进入"朋友圈"
2. 发布一条朋友圈（带图片）
3. 验证朋友圈发布成功
4. 好友查看朋友圈，验证能看到
5. 好友点赞
6. 验证点赞显示
7. 好友评论
8. 验证评论显示
9. 删除朋友圈
10. 验证朋友圈已删除

**预期结果**:

- [ ] 朋友圈发布成功
- [ ] 好友可见
- [ ] 点赞功能正常
- [ ] 评论功能正常
- [ ] 删除功能正常

---

### TC-11: WebSocket 连接

**优先级**: Critical
**前置条件**: 已登录用户

**测试步骤**:

1. 登录后验证 WebSocket 连接
2. 发送消息，验证实时接收
3. 断开网络
4. 恢复网络
5. 验证 WebSocket 自动重连
6. 验证离线期间的消息在重连后收到

**预期结果**:

- [ ] WebSocket 连接稳定
- [ ] 消息实时推送
- [ ] 自动重连正常
- [ ] 离线消息补推

---

### TC-12: Push 通知

**优先级**: Medium
**前置条件**: 移动端已登录，已配置 Push

**测试步骤**:

1. 在设置中开启通知
2. 锁定手机屏幕
3. 其他用户发送消息
4. 验证收到 Push 通知
5. 点击通知，验证打开应用并跳转到聊天
6. 在设置中关闭通知
7. 验证不再收到通知

**预期结果**:

- [ ] 通知功能正常
- [ ] 点击跳转正确
- [ ] 关闭通知功能正常

---

### TC-13: AI 功能（如配置）

**优先级**: Medium
**前置条件**: 已配置 AI provider 和 API key

**测试步骤**:

1. 进入"设置" -> "AI 设置"
2. 添加 AI API key
3. 验证 key 验证成功
4. 使用 AI 功能（如对话摘要）
5. 验证 AI 响应正确
6. 删除 API key
7. 验证 AI 功能不可用

**预期结果**:

- [ ] API key 管理正常
- [ ] AI 功能可用
- [ ] 删除后功能禁用

---

### TC-14: 多设备同步

**优先级**: High
**前置条件**: 同一用户在两个设备登录

**测试步骤**:

1. 设备 A 登录
2. 设备 B 登录同一账号
3. 在设备 A 发送消息
4. 验证设备 B 实时收到消息
5. 在设备 B 发送消息
6. 验证设备 A 实时收到消息
7. 在设备 A 退出登录
8. 验证设备 B 不受影响

**预期结果**:

- [ ] 多设备消息同步
- [ ] 实时推送正常
- [ ] 单设备退出不影响其他设备

---

### TC-15: 安全性验证

**优先级**: Critical
**前置条件**: 已登录用户

**测试步骤**:

1. 尝试访问旧 API 路径（如 `/user/profile`）
2. 验证返回 404/405
3. 尝试使用过期 token 访问 API
4. 验证返回 401
5. 查看错误响应
6. 验证不包含敏感信息（stack trace、密码等）
7. 在数据库中验证 E2EE 消息无明文

**预期结果**:

- [ ] 旧路径不可访问
- [ ] Token 验证正常
- [ ] 错误响应安全
- [ ] 无明文泄露
